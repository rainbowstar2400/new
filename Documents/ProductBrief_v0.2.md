# Product Brief v0.2

- 作成日: 2026-02-07
- プロダクト名（仮）: 自分専用秘書PWA
- 方針: オンラインMVP
- 参照: `ProductBrief_v0.1.md`

## 1. 背景（v0.1 -> v0.2）

- `v0.1` はローカル完結を前提としていた
- 現在の実装方針は `API + DB + Web Push + GPT要約スロット` を含むオンラインMVP
- そのため、ローカル専用前提のままでは仕様と実装が整合しない

## 2. 解決したい課題

- タスク登録そのものが面倒で、登録漏れが起きる
- 登録済みタスクを忘れる
- タスク以外の「覚えておきたいこと」も忘れる
- メモ/タスク/リマインドが複数アプリに分断され、運用が続かない

## 3. 対象ユーザー

- 初期: 作成者本人（自己運用）
- 将来: 同種課題を持つ個人ユーザー

## 4. MVP提供価値

- 「話しかける/書き込む」だけで、タスクまたはメモに登録できる
- 判定が曖昧なときは確認質問で誤保存を防ぐ
- 期日設定、時刻補完、オフセット調整まで会話で完結できる
- メモは `やりたいこと/アイデア/メモ（雑多）` へ分類される
- 通知で「忘れ防止」を支援する

## 5. MVPスコープ

- フロント: `Vite + React + TypeScript`
- API: `Node.js (Fastify)`
- DB: `PostgreSQL`
- 会話入力 + 自動分類（タスク/メモ）
- 期日3択 (`設定する/設定しない/後で設定する`)
- 日付のみ入力時の既定時刻補完 + `○/✕`
- `✕` 選択時のみ自然言語入力を追加受付
- オフセット解釈（`1時間前/30分前/前日` など）
- 対象曖昧時の `○/✕` 確認
- 要約スロット (`〇〇`) のみGPT生成 + フォールバック
- 匿名デバイスIDによるユーザー識別
- Web Push購読/配信、通知タップで対象タスクへ遷移

## 6. 非目的（MVPでやらない）

- メール認証
- 複数アカウント管理
- 高度な推薦
- 外部サービス連携（SNS/カレンダー双方向同期）

## 7. データ送信・プライバシー原則

- 送信は目的限定・最小化
- GPT送信は「ユーザー入力 + 確定済み構造化情報」のみ
- 未確定情報は送信しない
- Push送信は通知表示に必要な情報に限定
- 監査ログは最小限（入力・応答・時刻）

## 8. 成功指標（MVP）

- 5秒以内保存率: 80%以上
- タスク/メモ修正率: 20%以下
- メモ内修正率: 25%以下
- リマインド調整成功率: 95%以上
- 通知後10分以内確認率: 70%以上
